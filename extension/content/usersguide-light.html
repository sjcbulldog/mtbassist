2. üöÄ Installation & Setup

Prerequisites

  • Visual Studio Code 1.74.0 or later
  • Internet connection for initial setup and content downloads
  • Windows, macOS, or Linux operating system

Installation Steps

  1. Open VSCode and navigate to the Extensions panel (Ctrl+Shift+X)
  2. Search for "ModusToolbox Assistant"
  3. Click "Install" on the extension
  4. Restart VSCode when prompted
  5. The extension will appear in the Activity Bar with the ModusToolbox icon

‚ÑπÔ∏è First Launch

On first launch, the extension will check for manifest data and may prompt you to configure your internet connection settings if you're behind a corporate firewall.

4. üõ†Ô∏è Creating Projects

The Create Project wizard guides you through a 4-step process to set up new ModusToolbox projects.

Step 1: Project Information

  1. Project Name: Enter a unique project name (no spaces allowed)
  2. Project Location: Choose or browse to your desired project directory
  3. The extension will show a preview of the full project path
  4. Click "Next: Select BSP" to continue

‚ö†Ô∏è Naming Restrictions

Project names and paths cannot contain spaces. Use hyphens (-) or underscores (_) instead.

Step 2: BSP Selection

Choose your Board Support Package based on your target hardware:

Connected Development Kits

  • The extension automatically detects connected development kits
  • Click "Refresh" to scan for newly connected devices
  • Select from available connected kits for automatic BSP selection

Manual BSP Selection

  1. Choose a BSP Category from the dropdown
  2. Select a specific Board Support Package
  3. Hover over BSPs to see detailed descriptions
  4. The description panel shows device specifications and connectivity options

Step 3: Example Selection

  1. Choose an Example Category (e.g., "Getting Started", "Connectivity", "Security")
  2. Select a specific Code Example to use as your project template
  3. Review the example description to understand its functionality
  4. Click "Review & Create" to proceed

Step 4: Review & Create

  • Review all your selections in the project summary
  • Verify the project path and configuration
  • Click "Create Project" to begin the creation process
  • Monitor progress with the real-time progress bar
  • Once complete, click "Load Project" to open in VSCode

‚ÑπÔ∏è Manifest Data Required

Project creation requires manifest data to be available. If you see a "Waiting on manifest data" message, check your internet connection or Local Content Storage configuration.

5. üì¶ Local Content Storage

The Local Content Storage feature allows you to download and manage BSPs locally for offline development.

Interface Overview

The Local Content Storage interface consists of two main panels:

üìã Not Available Locally

Lists all BSPs available for download from the remote repository. Use the filter box to search for specific packages.

üíæ Available Locally

Shows BSPs already downloaded and stored locally on your machine for offline use.

Managing BSPs

Adding BSPs to Local Storage

  • Drag & Drop: Drag BSPs from the left panel to the right panel
  • Add All: Click the "Add All" button to download all available BSPs
  • Individual Selection: Select specific BSPs for targeted downloads

Removing BSPs from Local Storage

  • Drag & Drop: Drag BSPs from the right panel back to the left panel
  • Remove All: Click the "Remove All" button to clear local storage

Advanced Features

Show Changes Filter

  • Check "Show Changes" to view only BSPs with pending add/remove operations
  • When enabled, the filter box is automatically disabled and cleared
  • Use this to review changes before applying them

Apply and Revert

  • Apply: Commits all pending changes (downloads/deletions)
  • Revert: Cancels all pending changes and returns to the previous state
  • The "Show Changes" filter is automatically unchecked after applying changes

‚ö†Ô∏è Storage Space

BSPs can be large (several hundred MB each). Ensure you have adequate disk space before downloading multiple packages.

  • Wait for completion before using ModusToolbox features

Installation Controls

  • Browse Button: Automatically selects the custom directory card

9. üîß Troubleshooting

Common Issues

Manifest Data Not Available

  1. Check your internet connection
  2. Verify firewall settings allow VSCode extension communication
  3. Try refreshing the extension by reloading VSCode
  4. Check Local Content Storage configuration

Project Creation Fails

  • Ensure project name and path don't contain spaces
  • Verify the target directory is writable
  • Check that BSP and example selections are valid
  • Ensure sufficient disk space for project creation

BSP Download Issues

  • Verify internet connectivity
  • Check available disk space for local storage
  • Try downloading BSPs individually instead of using "Add All"
  • Clear browser cache if using web-based authentication

Software Installation Problems

  • Ensure installation path doesn't contain spaces
  • Run VSCode as administrator (Windows) if permission errors occur
  • Check that installation directory is writable
  • Verify Infineon account credentials are correct

Performance Optimization

Improving Extension Performance

  • Close unused project tabs in Application Status
  • Limit the number of BSPs stored locally
  • Use filtering in Local Content Storage to find specific packages
  • Restart VSCode periodically to clear cached data

Getting Help

  • Check the VSCode Developer Console for error messages
  • Review extension logs in the Output panel
  • Contact Infineon support through official channels
  • Report issues through the VSCode extension marketplace
